# DataOS 数据平台架构文档 v3.0

> 基于《MES 高质量结构化数据集解决方案》重构  
> 去除项目管理概念，聚焦数据接入→标准→清洗→建模→数据集全链路  
> 最后更新: 2026-06-09

---

## 一、项目定位

DataOS 是面向多客户 MES 项目的数据集生产平台。核心目标是将不同客户项目的 MES 数据，通过统一的数据标准、字段映射、清洗规则和模型设计，转化为可复用、可验收、可发布的高质量结构化数据集。

**一句话**: 把多项目的差异化原始数据，变成公司级的标准化数据资产。

```
客户MES项目A ──┐
客户MES项目B ──┼──→ DataOS ──→ 标准化数据集 ──→ Directus API / 文件导出
客户MES项目C ──┘                    │
                              RelOS 关系图
```

### 2.2 10 个模块清单

| # | 模块 | 定位 | 输入 | 输出 |

| 2 | **数据接入** | 客户MES库连接与数据采集 | 数据库连接信息 | 原始数据(Bronze)/元数据清单/同步日志 |
| 3 | **数据标准** | 公司级字段/编码标准管理 | 原始字段清单 | 标准字段库/字段映射/编码字典 |

## 五、数据标准模型（双范围隔离）

DataOS 采用**公司级共享 + 项目级隔离**的双层模型：

```
公司级 (scope=global, 所有项目共享)      项目级 (scope=project, 项目隔离)
─────────────────────────────────      ─────────────────────────────

标准字段库 (data_standards)            数据源 (datasources)
  WORK_ORDER_CODE                        客户A MES库
  EMPLOYEE_NAME                          客户B MES库
  MRL_CODE                              

编码字典 (code_dictionaries)           Pipeline (cleaning_pipelines)
  工单状态: 0→未发布                     数据清洗 → 项目A
                                       数据清洗 → 项目B

字段映射 (field_mappings)              质量规则 (quality_rules)
  项目A: ORDER_NO → WORK_ORDER_CODE      非空检查项目A
  项目B: OID     → WORK_ORDER_CODE      非空检查项目B

主题域 (data_domains)                  业务过程 (business_processes)
  生产调度/生产执行/质量管理              项目A订单创建 → 归属生产调度域
                                       项目B派工执行 → 归属生产执行域
```

### 隔离规则

| 对象 | 作用域 | 规则 |
|------|:--:|------|
| 数据源连接 | project | 每个项目独立配置, 不可跨项目访问 |
| 原始数据(Bronze) | project | 按 `projects/{pid}/datasources/{dsid}/` 分区 |
| 清洗数据(Silver) | project | 按 `projects/{pid}/pipelines/{name}/` 分区 |
| Gold 表 | project | 通过 target_table 命名区分 |
| 标准字段 | global/project | global 全平台可见, project 项目内可见 |
| 编码字典 | global/project | 同上 |
| 字段映射 | project | 每个项目维护自己的映射关系 |
| 数据域 | global/project | global 域全平台可见 |
| 业务过程 | project | 项目内可见 |
| 数据集 | project | 项目内生成, 可跨项目发布 |
| 标签 | global | 全平台共享 |

## 六、关键设计决策

| # | 决策 | 说明 |
|---|------|------|
| 1 | 标准先行 | 先建公司级标准字段库, 再接入项目数据, 确保统一 |
| 2 | 字段映射是核心 | 不同源字段→统一标准字段, 带置信度评分 |
| 3 | 清洗规则可复用 | 规则模板化, 不同项目复用同一套规则 |
| 4 | 模型驱动建表 | 建模完成后自动生成 DDL, 一键创建物理表 |
| 5 | 数据集版本化 | 每次生成新版本, 保留历史, 可回滚 |
| 6 | 验收即发布 | 质量校验通过后自动发布, 不通过驳回整改 |
| 7 | Directus 零代码 API | Gold 表自动被发现, 无需额外开发 |
| 8 | 全链路审计 | 每次操作记录 AuditLog, 可追溯 |
